# image-to-pptx

## Goal

把每张页面图片重建成可编辑 `.pptx`：非文字视觉元素拆成独立透明 PNG；所有可读文字都重新创建为可编辑文本框；表格必须优先创建为 PowerPoint 原生表格；最终字体统一使用 `微软雅黑` / `Microsoft YaHei`。

## Workflow

1. **识别页面对象**：列出文字、表格、图标、插图、图表、背景、框图、箭头和分隔线。
2. **拆分 PNG**：只裁非文字视觉元素。每个元素单独 PNG，透明背景，裁完整，保留安全边距和原比例。
3. **彻底去文字**：任何进入 PNG 的区域都必须先剔除可读文字；若原图区域含文字，改为“无文字图片 + 可编辑文本框”组合。
4. **表格原生化**：判断为表格时，用 PowerPoint table 输出，不裁成图片。可用 `scripts/native_table_pptx.py` 生成/参考原生表格。
5. **重建 PPTX**：按原位置放回 PNG、形状、表格和文字。文字框必须在原框图内。
6. **QA 调整**：检查图片是否裁完整、文字是否仍残留在 PNG、文本是否超框；必要时缩小字号、调行距、扩大文本框或微调元素尺寸。

## Hard Rules

- 不允许整页截图作为背景。
- 不允许把可读文字留在 PNG 中。
- 不允许裁断图标、实验图、图表、边框或箭头。
- 不允许合并多个独立视觉元素，除非它们天然是一个不可拆图组。
- 所有可编辑文字使用 `微软雅黑`，字号宁可偏小，不可溢出原框。
- 表格内容必须是可编辑单元格文字；表格线、底色、强调行列用原生 table 样式或形状实现。
- 复杂图表内无法安全 OCR 的极小标签可暂留为图片，但必须在最终说明中列为 known limit。

## Required Checks

- PNG 资产：透明背景、无文字残留、无裁断、边缘有 2-8 px 安全边距。
- 文字：源图可读文字均已转成文本框或表格单元格；字体为微软雅黑；没有超框或遮挡。
- 表格：能识别行列结构时必须是原生 PPT table。
- 布局：每个 PNG、文本框、表格都在原始框图或对应区域内，比例和相对位置接近源图。
- 输出：交付 `.pptx`、拆分 PNG 资产、对象/资产映射清单和 QA 备注。
