
# Precision Rebuild Prompt

请将用户提供的 PPT 页面图片重建成高保真、可编辑 PowerPoint。

## 第一步：页面拆分

- 将每页所有视觉元素拆分为单独 PNG。
- 每个非文字视觉元素单独一张 PNG，不合并元素。
- PNG 必须透明背景，裁掉多余透明区域，但保留 2-8 px 安全边距。
- 裁剪必须完整：图标、箭头、实验图、图表、边框、阴影、插图都不能被切断。
- PNG 不得包含任何可读文字；如果源区域含文字，先把文字移除或把该区域拆成“无文字视觉底图 + 可编辑文本框”。
- 保持元素原始比例、相对位置、大小和层级。

## 第二步：文字识别与去重建

- 识别页面中所有可读文字，包括标题、副标题、段落、项目符号、坐标轴标签、表头、表格单元格、脚注和小注释。
- 所有识别出的文字必须创建为独立可编辑文本框或表格单元格。
- 最终字体统一使用 `微软雅黑` / `Microsoft YaHei`。
- 文本框必须落在原始文字所在框图内；如果文字超框，优先缩小字号、压缩行距、适度扩大文本框，不要覆盖其他元素。
- 若自动 OCR 不可靠，必须人工目视补齐明显可读文字；不能因为 OCR 漏识别就忽略。

## 第三步：表格处理

- 如果页面区域具有清晰行列结构、表头、单元格或网格线，判断为表格。
- 表格必须用 PowerPoint 原生 table 输出，不要裁成图片。
- 表格线、底色、强调行、强调列、数字颜色都要尽量复刻。
- 表格文字写入单元格，字体为微软雅黑。
- 可参考或调用 `scripts/native_table_pptx.py` 生成原生表格。

## 第四步：PPTX 还原

- 使用拆出的 PNG、原生形状、原生表格和可编辑文本框重建完整页面。
- 所有 PNG 按原始页面坐标放回，保持原比例和位置关系。
- 简单线条、矩形、圆角框、分隔线、箭头优先用 PPT 原生形状。
- 复杂插图、实验图、图标、分子图、设备图等作为独立 PNG。
- 不使用整页截图作为背景。

## 第五步：最终检查

- 检查每个 PNG 是否完整、透明、无文字残留。
- 检查每段文字是否已变成可编辑文本，是否仍在原框图内。
- 检查表格是否为原生 table，单元格内容可编辑。
- 检查是否有文字溢出、遮挡、跑位或字号过大。
- 若发现问题，修正后重新导出 PPTX。

最终输出：

- 一个 `.pptx` 文件。
- 拆分 PNG 资产。
- 资产与页面位置映射清单。
- 简短 QA 备注，说明无法完全可编辑化的极小标签或复杂图内文字。
